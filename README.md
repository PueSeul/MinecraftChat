# Java Chat

Minecraft Java Edition **1.8.9부터 26.2까지**의 서버에 안드로이드에서 접속해 채팅을 읽고 보내며, 접속자 탭 목록을 확인하는 앱입니다. 서버 플러그인을 사용하지 않고 일반 Minecraft Java 클라이언트처럼 직접 접속합니다. 여기서 `26.2`는 `1.26.2`가 아닌 Minecraft의 새 버전명입니다.

[Java Chat v1.0 APK 바로 다운로드](https://github.com/PueSeul/JavaChat-Android/releases/download/v1.0/JavaChat-v1.0.apk)

## 현재 구현

- 여러 서버 저장, 추가, 편집, 삭제(서버에는 주소·포트·버전만 저장)
- 서버별 Minecraft 버전 선택 및 저장(선택창에는 내부 프로토콜 번호 없이 버전명만 표시)
- Java 1.8.9–26.2의 65개 공식 지원 릴리스 선택지
- 26.2 클라이언트 프로토콜과 ViaVersion 기반 하위 버전 변환
- 첫 화면의 전역 온라인/오프라인 버튼으로 모든 서버의 접속 방식 전환
- Microsoft 기기 코드 로그인 및 정품 서버 접속
- Android Keystore AES-GCM으로 Microsoft 세션 암호화 저장
- 오프라인 닉네임 접속(`online-mode=false` 서버 전용)
- 일반 채팅과 `/` 명령어 전송, 시스템 메시지 표시
- 붙여넣기나 키보드 입력에 포함된 줄바꿈·제어문자를 안전한 공백으로 정규화해 서버 강제 종료 방지
- 서버에 명령어 자동완성을 요청하고 입력창 위에 선택 목록 표시
- 26.2 서명 채팅 세션 및 RSA-SHA256 메시지·명령어 인자 서명, 구버전 채팅 변환
- 서버 접속자 탭 목록과 지연 시간 표시
- 접속자 닉네임 왼쪽에 32dp 크기로 스킨의 얼굴과 모자 레이어를 선명한 최근접 보간으로 표시(메모리 캐시 및 기본 얼굴 지원)
- 서버 패킷에 스킨이 없으면 원래 닉네임으로 Mojang 프로필을 조회해 정품 스킨 표시
- 포그라운드 서비스로 화면이 바뀌어도 연결 유지
- 확장한 연결 알림의 **서버 나가기** 버튼으로 즉시 연결 종료
- 저장한 서버마다 Minecraft 서버 목록 핑으로 온라인/오프라인, 접속자 수와 지연 시간 표시
- 첫 화면에서 GitHub의 최신 정식 릴리스를 확인하고, 새 APK의 해시·패키지·버전·서명을 검증한 뒤 Android 업데이트 설치 화면 실행
- Android 호환 SRV DNS 조회로 `_minecraft._tcp` 주소 지원
- Android 15/16의 상단 상태 표시줄, 하단 내비게이션 바와 키보드 인셋 대응
- 서버 편집 화면의 버전 선택·저장·삭제 버튼 사이 간격 조정
- 접속 중 뒤로가기 또는 종료 버튼을 누르면 연결 종료 확인
- Microsoft 로그인 실패 원인을 계정 없음, DNS, 시간 초과 등으로 구분해 안내
- Minecraft 인증서 요청을 JSON으로 전송해 `415 Unsupported Media Type` 오류 대응
- Java 26.2 공식 authlib 필드에 맞춘 프로필 공개키 서명 전송
- 서버가 Microsoft 계정과 다른 오프라인 UUID를 부여하면 공식 클라이언트처럼 호환되지 않는 프로필 공개키를 보내지 않음
- 버전별 `ProtocolAdapter` 구조로 이후 신규 버전 추가 가능

## 설치

`JavaChat-v1.0.apk`를 휴대폰으로 옮겨 설치합니다. GitHub에서 직접 배포하는 APK이므로 Android에서 이 출처의 앱 설치 허용을 요청할 수 있습니다.

ADB가 연결된 경우:

```powershell
adb install -r JavaChat-v1.0.apk
```

## 사용

1. 첫 화면에서 모든 서버에 적용할 **온라인** 또는 **오프라인** 접속 방식을 선택합니다.
2. 온라인 방식은 **Microsoft 로그인**을 눌러 브라우저의 기기 코드 로그인을 완료합니다.
3. 오프라인 방식은 첫 화면에 닉네임을 입력하고 저장합니다. 서버가 `online-mode=false`일 때만 접속할 수 있습니다.
4. **서버 추가**에서 이름, 주소, 포트와 버전만 저장합니다.
5. 저장한 서버의 **접속**을 누르면 현재 선택한 전역 접속 방식이 사용됩니다.
6. 채팅 입력란에 `/`로 시작하는 내용을 입력하면 서버가 제공하는 명령어 제안이 표시됩니다. 제안을 누르면 해당 부분이 입력란에 적용됩니다.
7. 접속 화면에서 뒤로가기나 **종료**를 누르면 확인창에서 연결 종료 여부를 선택할 수 있습니다. 알림을 확장해 **서버 나가기**를 눌러도 즉시 종료됩니다.
8. 첫 화면의 **업데이트 확인**을 누르면 최신 GitHub 정식 릴리스를 확인합니다. 새 버전이 있으면 **업데이트**를 눌러 APK 다운로드와 검증을 진행하고 Android 설치 화면에서 최종 확인합니다.

이후 릴리스의 앱 내 자동 업데이트를 유지하려면 GitHub 태그와 APK 파일명을 각각 `v1.1`, `JavaChat-v1.1.apk` 같은 형식으로 맞추고 기존과 같은 앱 서명 ID를 사용해야 합니다.

## 소스 빌드

요구 사항은 JDK 17과 Android SDK입니다. `local.properties.example`을 `local.properties`로 복사한 뒤 자신의 SDK 경로로 바꿉니다.

```powershell
.\gradlew.bat :app:testDebugUnitTest :app:lintRelease :app:assembleRelease
```

APK는 `app/build/outputs/apk/release/app-release.apk`에 생성됩니다. 최소 Android 버전은 8.0(API 26), 대상 SDK는 API 37입니다.

## 테스트 범위와 제한

- 자동 테스트, Android Lint, 릴리스 APK 빌드와 APK v2 서명 검증을 통과했습니다.
- 버전 목록의 65개 항목 모두가 26.2에서 해당 서버 버전으로 이어지는 완전한 변환 경로를 갖는지 자동 검사합니다.
- 공식 서버 1.8.9, 1.12.2, 1.16.5, 1.19.4, 1.21.11에서 오프라인 로그인, 월드 입장, 일반 채팅 전송, `/list` 결과 수신, `/he` 자동완성과 접속자 목록 수신을 통합 테스트했습니다.
- 구버전은 월드 입장 좌표와 텔레포트 확인이 완료된 뒤에만 채팅 입력이 활성화됩니다.
- 로컬 26.2 테스트 서버를 이용한 서버 목록 핑 자동 통합 테스트에서 온라인 여부, `3/20` 접속자 수와 버전 응답을 확인했습니다.
- 명령어 트리의 서명 대상 인자 추출, RSA 서명 명령어 패킷, 프로필 UUID 정책과 여러 종류의 줄바꿈 정규화를 단위 테스트로 확인했습니다.
- 플레이어 프로필의 스킨 URL 추출과 Mojang 텍스처 CDN 제한, 모자 표시 설정 전달을 단위 테스트로 확인했습니다.
- Samsung SM-S908N에서 실제 원격 Java 26.2 서버에 Microsoft 계정 방식으로 접속해 공개키 오류 없이 연결 유지와 접속자 목록 수신을 확인했습니다. 서버가 계정과 다른 오프라인 UUID를 부여한 경우 프로필 키를 생략하는 동작도 확인했습니다.
- ADB로 입력창에 실제 2줄 메시지를 만든 뒤 전송해 서버 응답, 연결 유지와 입력창 초기화를 확인했습니다. 프로토콜에는 줄바꿈 대신 안전한 공백이 전달됩니다.
- 실제 접속자 정보에 스킨이 없는 서버에서 Mojang 프로필 조회로 PueSeul의 스킨 얼굴이 표시되는 것을 확인했습니다. 머리 이미지 크기와 서버 편집 화면의 버튼 간격도 SM-S908N 실기기에서 확인했습니다.
- 이 앱은 월드 렌더링이나 이동을 하지 않는 헤드리스 클라이언트입니다. 서버의 AFK 정책, 봇 방지, 커스텀 인증 플러그인 등에 의해 퇴장될 수 있습니다.
- 서버가 플레이어 정보에 스킨 텍스처를 제공하지 않거나 이미지 다운로드에 실패하면 접속자 목록에 기본 얼굴이 표시됩니다.
- 명령어 자동완성 내용은 접속한 서버가 제공하며, 서버 권한에 따라 보이는 명령어가 달라집니다.
- 오프라인 닉네임으로는 `online-mode=true` 서버에 로그인할 수 없습니다.
- Microsoft 계정에는 Minecraft Java Edition 사용 권한이 있어야 합니다.
- 처음 실행할 때 기존 서버별 인증 설정 중 첫 번째 서버의 값을 전역 설정으로 한 번 이전합니다. 이후에는 서버별 인증 값이 사용되지 않습니다.
- Microsoft 로그인과 정품 인증 서버 접속에는 유효한 계정이 필요합니다. Java 26.2 공개키 서명 선택과 UUID 일치 정책은 공식 26.2 클라이언트의 `authlib 9.0.75` 처리, 단위 테스트와 실제 서버 접속으로 검증했습니다.
- 1.8 계열은 모든 패치가 같은 프로토콜 47을 사용하므로 선택창의 `Java 1.8.9` 항목으로 1.8.x 서버에 접속합니다.

## 버전 변환 및 확장 지점

`ProtocolRegistry`에 Java 1.8.9–26.2의 릴리스명과 내부 프로토콜을 등록합니다. 기본 26.2 구현은 GeyserMC MCProtocolLib의 `feature/26.2` 브랜치, 커밋 `b77d50c33a17601e6b71078fe9b6de7d98b4ff13`을 프로젝트 내부 `mcprotocol` 모듈로 고정했고, 이전 버전 연결은 ViaVersion 5.11.1-SNAPSHOT의 클라이언트 측 변환 파이프라인을 사용합니다. `viaversion-android` 모듈은 Android에 없는 데스크톱 `Runtime.version()` 의존 클래스 세 곳을 ViaVersion 자체의 기존 호환 구현으로 대체합니다. 다음 신규 버전도 저장 형식과 화면을 유지한 채 `ProtocolRegistry` 및 어댑터를 확장할 수 있습니다.
